WHY THE APP SHOWED HISTORY LOADED = 0

GitHub-hosted runners are ephemeral/clean VMs. For a full-universe, no-cut build, that means the runner has no persistent history store between jobs.
So every workflow run would have to redownload and rebuild everything again, which is too heavy and unreliable.

USE THIS INSTEAD
1. Run: run_daily_builder.bat
   or: python build_daily_local.py
2. Wait until it finishes building scanner snapshots.
3. Commit only these changed files to GitHub:
   - scanner_pro/data/universes/*.csv
   - scanner_pro/data/scans/*.csv
   - scanner_pro/data/scans/*.json
   - scanner_pro/data/macro/latest_snapshot.json
   - macroregime_pro/.cache/latest_snapshot.json
   - macroregime_pro/.cache/latest_snapshot_manifest.json
4. Push.
5. Refresh Streamlit.

DO NOT COMMIT
- scanner_pro/data/history/*
- macroregime_pro/data/history/*
- cache files not listed above
